Implementation guardrails

Boundaries that keep the rebuild incremental, compatible, and reviewable.

Compatibility first

No HTTP, RPC, WebSocket, event, scope, or deep-link identifier is renamed in 0.29.0. Optional schema-backed additions require old/new compatibility fixtures.

One state system

The workspace package is React-independent and preserves Effect Atom plus the environment registry. Clients own navigation, rendering, storage adapters, and native modules.

Explicit capabilities

Feature code queries narrow services for persistence, notifications, links, clipboard, window/SSH, terminal, review, and composer operations—never platform globals.

Failure is a first-class state

Loading, reconnecting, stale, offline, auth-expired, version-skew, unsupported, retryable, and terminal failures retain useful context without exposing credentials.

Platform-native presentation

Web uses compact three-region density and keyboard focus. iOS uses 44-point targets, Dynamic Type, VoiceOver, native stack/sheets, haptics, and reduced motion.

Release boundary

Only the final validated 0.29.0 is installable. No push to main, publishing, DMG distribution, or iOS submission occurs without separate approval.
